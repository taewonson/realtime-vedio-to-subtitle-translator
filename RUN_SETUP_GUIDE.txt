[프로그램 실행 조건 및 설정 가이드]

1) 개요
- 이 프로젝트는 Chrome 확장 프로그램이 YouTube 재생 상태를 로컬 Python 앱으로 전송하고,
  Python 앱이 Google Cloud Speech-to-Text + Translation으로 자막을 생성/번역하여 Tkinter UI에 표시합니다.

2) 필수 소프트웨어
- Windows 10/11
- Python 3.10+ (권장: 3.11 이상)
- Google Cloud 서비스 계정 키(JSON)
- ffmpeg 실행 파일 (PATH 등록 또는 FFMPEG_PATH 지정)
- Chrome 브라우저

3) Python 의존성 설치
- 작업 폴더 이동:
  cd google_cloud_sync_app
- 설치:
  pip install -r requirements.txt

requirements.txt 주요 항목:
- google-cloud-speech
- google-cloud-translate
- yt-dlp

4) Google Cloud 준비
- 프로젝트 생성/선택
- Speech-to-Text API 활성화
- Cloud Translation API 활성화
- 서비스 계정 생성 후 JSON 키 발급
- 서비스 계정에 필요한 권한 부여

5) 환경 변수 설정 (Windows CMD 예시)
- set GOOGLE_CLOUD_PROJECT=your-project-id
- set GOOGLE_APPLICATION_CREDENTIALS=C:\path\to\service-account.json
- set SOURCE_LANGUAGE_CODE=en-US
- set TARGET_LANGUAGE_CODE=ko
- set GOOGLE_TRANSLATE_LOCATION=global

5-1) .env 파일로 간편 설정 (권장)
- 위치: google_cloud_sync_app\.env
- 예시 파일: google_cloud_sync_app\.env.example
- 사용 방법:
  1. google_cloud_sync_app\.env.example 파일을 복사해서
     google_cloud_sync_app\.env 파일 생성(.env.example은 인식하지 않기 때문에 .env를 생성해야함)
  2. 프로젝트 값/언어 코드를 본인 환경에 맞게 수정
  3. 이후 main.py 실행 시 .env 값을 자동 로드

주의:
- OS에 이미 설정된 환경변수 값이 있으면 .env 값보다 OS 값이 우선합니다.
- 인증 키 경로(GOOGLE_APPLICATION_CREDENTIALS)는 보안상 시스템 환경변수로
  설정하는 방식을 권장합니다.

참고:
- SOURCE_LANGUAGE_CODE는 STT 인식 언어(BCP-47, 예: en-US)
- TARGET_LANGUAGE_CODE는 번역 목표 언어(예: ko, en, ja)

6) ffmpeg 설정
방법 A) 시스템 PATH에 ffmpeg 등록
방법 B) 환경 변수 직접 지정
- set FFMPEG_PATH=C:\ffmpeg\bin\ffmpeg.exe

확인:
- ffmpeg -version

7) Chrome 확장 프로그램 설치
- Chrome 주소창에 chrome://extensions 입력
- 우상단 "개발자 모드" 활성화
- "압축해제된 확장 프로그램을 로드" 클릭
- 프로젝트의 google_cloud_extension 폴더 선택

8) 애플리케이션 실행 순서
- 터미널에서:
  cd google_cloud_sync_app
  python main.py
- 앱이 실행되면 상태 메시지에 sync 서버 시작 문구가 표시됩니다.
- Chrome에서 YouTube 영상을 재생하면 자막이 Tkinter 창에 표시됩니다.

9) 동작 원리 요약
- 확장 프로그램(content.js): 주기적으로 재생 시간/URL/재생속도 전송
- browser_sync/server.py: /sync 수신 후 BrowserState 큐 적재
- media/*: yt-dlp로 오디오 URL 해석 후 ffmpeg로 PCM 스트림 생성
- stt/*: Google Speech 스트리밍 인식
- subtitle/*: 타임라인 기준 자막 큐 저장/조회
- translate/*: 최종 자막만 번역 API 호출
- ui/*: 현재 자막/상태 표시

[최근 변경 반영]
- STT 결과/상태 큐 소비 지점이 단일화되었습니다.
  기존: 파이프라인 스레드 + UI 폴링 스레드 양쪽에서 poll
  변경: UI 폴링(_poll)에서만 poll 수행
  효과: 중복 소비나 레이스 컨디션 가능성 감소, 자막 처리 일관성 향상

10) 자주 발생하는 문제
- GOOGLE_CLOUD_PROJECT 누락:
  프로그램 시작 시 RuntimeError 발생 -> 환경 변수 설정 확인
- 인증 오류(권한/키 경로):
  GOOGLE_APPLICATION_CREDENTIALS 경로 및 키 파일 권한 확인
- ffmpeg 미설치:
  ffmpeg 경로 확인 또는 FFMPEG_PATH 설정
- 로컬 서버 연결 실패:
  확장 프로그램이 보내는 주소(127.0.0.1:8765)와 앱 실행 여부 확인
- YouTube 오디오 URL 해석 실패:
  yt-dlp 최신화 및 영상 접근 가능 여부 확인
  pip install -U yt-dlp

11) 운영 팁
- 긴 영상에서 메모리 사용량은 timeline_store의 prune_old 로직으로 제한됩니다.
- seek(탐색)나 URL 변경 시 파이프라인이 자동 재시작되도록 설계되어 있습니다.
- 번역 호출은 final STT 결과에만 수행되어 비용과 지연을 줄입니다.
- 실행 시에는 Python 앱(main.py)과 Chrome 확장 프로그램이 동시에 활성 상태여야
  실시간 동기화가 정상 동작합니다.
